  • Comtent Strategy
    • Content is all that matters.

      Anyone is here on your page just for content. It is one which matters. Rest is secondary.

    • Create user personas

      It will direct you to what are your audiences. It is a very important step in content strategy. You will find it helpful at later stages. You may create imaginary personas but it would be much better if you are able to involve real audience.

    • Content audit

      Audit the content which is supposed to be on your website. Gather and classify them in term on appearance, location, hierarchy and importance. It is advisable to involve a number of stake holders(user personas) as more as possible.

    • Content modeling

      Content modeling helps you to identify the redundancy and modularity of contents on your website.

    • Content Priority hierarchy

      Determine on what priority the specific content should be on your website. For eg, Home page is having high priority than legel page. So It should be more important than the latter one.

  • User Testing
    • Card Sorting

      Ask users to idenitfy and sort the website content in hierarchy and architecture they find easy. This can be used to design the contents, prioritise and build navigation. Involve as much as random census you can have and get the mean to fit the requirement.

    • Testing wireframes and interaction patterns

      In this, we ask user to identify the link between all the wireframes. Make sure that they are given minimum information and let them arrange the wireframes as per their understanding. These wireframes should have same information which the website is gonna have. This will impersonate them as they are accessing exact website. Ask their issues, suggestions and how much they understood it.

  • Test & Revision Phase
    • Accessibility Testing
      • Achecker Accessibility Check
      • Try to access your website using keyboard only.
      • Disable all styles and JS and read from bottom. It should have natural hierarchy
      • Ask real users to test it. They face it more natural than you.
    • Break and edge-case testing
      • Disable one or more stylesheet or JS and see how it affects your website.
      • Try to access your website on bad internet connection.
      • The user should be notified when anything breaks just because of any issue like poor internet connection.
      • Check all validations and limits on height width of controls.
    • End User Testing
      • Ask real users to test this website.
      • Choose random users from multiple subsets and ask them to test it.
      • Ask user to test on their own devices. It will give more exact feedback.
      • You can ask users to access this website on multiple devices simultaneously. This will guide them to provide the actual difference.
      • Just provide minimum instruction to user. It shouls be in form of a module which should be common to all and ask users to try to navigate throught each corner by their own.
